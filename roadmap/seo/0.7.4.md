# 0.7.4 - Page Metadata

## Goal

Add metadata to remaining pages: blog list, CV, and homepage.

## Deliverables

- [ ] Blog list page metadata
- [ ] CV page metadata
- [ ] Homepage metadata
- [ ] Consistent OG images across pages

## Implementation

### Blog List Page

```typescript
// apps/web/app/(main)/blog/page.tsx
import type { Metadata } from 'next';

export const metadata: Metadata = {
  title: 'Blog',
  description: 'Thoughts, stories, and ideas from different perspectives.',
  openGraph: {
    title: 'Blog | sbozh.me',
    description: 'Thoughts, stories, and ideas from different perspectives.',
    type: 'website',
    images: ['/og/blog.png'],
  },
  twitter: {
    card: 'summary_large_image',
    title: 'Blog | sbozh.me',
    description: 'Thoughts, stories, and ideas from different perspectives.',
    images: ['/og/blog.png'],
  },
};
```

### Static OG Images

Create static OG images for pages without dynamic content:

```
apps/web/public/og/
├── home.png      # Homepage OG image
├── blog.png      # Blog list OG image
├── cv.png        # CV page OG image
└── default.png   # Fallback OG image
```

These can be:
1. Designed manually (Figma/Canva)
2. Generated via API route similar to blog posts
3. Simple branded images with page title
